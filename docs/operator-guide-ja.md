# NeNe Invoice 運用ガイド（日本語）

NeNe Invoice の共有ホスティング（Tier A）向けインストール・運用手順書。

---

## 1. 動作要件

| 項目 | 要件 |
|---|---|
| PHP | 8.4 以上 |
| PHP 拡張 | pdo, pdo_mysql, mbstring, openssl, json |
| データベース | MySQL 5.7 以上 / MariaDB 10.5 以上 |
| Web サーバー | Apache (.htaccess 対応) または Nginx |
| ディスク | 50MB 以上（vendor 含む） |

> **注:** Xserver・さくらインターネット・ロリポップなど主要な共有ホスティングで動作します。

---

## 2. インストール手順

### 2-1. ZIP を展開してアップロード

1. リリース ZIP（`nene-invoice-x.x.x.zip`）を展開する
2. 展開したすべてのファイルをサーバーの `public_html/` 以下にアップロード
   - 例: `public_html/invoice/` に配置する場合はその中に展開したファイルを置く

### 2-2. パーミッション設定

```
chmod 755 public_html/
chmod 644 public_html/*.php
chmod 755 var/          # なければ作成して 755
```

### 2-3. Web インストーラーを実行

ブラウザで `https://your-domain.example/install.php` にアクセスします。

**ステップ 1: データベース設定**

| 項目 | 内容 |
|---|---|
| ホスト | 通常 `localhost`（ホスティング会社に確認） |
| ポート | 通常 `3306` |
| データベース名 | 事前に作成した DB 名 |
| ユーザー名 | DB ユーザー名 |
| パスワード | DB パスワード |

「接続テスト & スキーマ適用」ボタンをクリックするとテーブルが自動作成されます。

**ステップ 2: 管理者設定**

| 項目 | 内容 |
|---|---|
| 会社名 | 発行者（自社）の名称 |
| 管理者メール | ログインに使うメールアドレス |
| パスワード | 8 文字以上 |

「インストール実行」でセットアップ完了です。

### 2-4. インストーラーを削除（重要）

```bash
# SSH が使える場合
rm public_html/install.php
```

SSH が使えない場合は FTP クライアントで `install.php` を削除してください。  
または `.htaccess` に以下を追加してアクセスを禁止します。

```apache
<Files "install.php">
    Require all denied
</Files>
```

## 3. 初期設定

### 3-1. ログイン

`https://your-domain.example/admin/` にアクセスしてインストール時に設定したメール・パスワードでログインします。

### 3-2. 会社設定（登録番号の設定）

1. 上部ナビの「設定」をクリック
2. **登録番号**（インボイス制度の適格請求書発行事業者番号、`T` + 13 桁）を入力
3. 銀行振込先情報を入力
4. 「保存する」をクリック

> 登録番号を設定しないと「適格請求書（インボイス）」として発行できません。

### 3-3. 取引先を登録

1. 「取引先」→「取引先を作成」
2. 社名・担当者名・メールアドレス・請求書送付先住所を入力
3. 「作成」をクリック

---

## 4. 基本的な使い方

### 見積書の作成

1. 「見積書」→「見積書を作成」
2. 取引先・明細（品目・数量・単価・税率）を入力
3. 「作成する」で下書き保存
4. 詳細画面で「送付する」→ 先方承認後「承認する」→「請求書に変換する」

### 請求書の発行

1. 「請求書」→ 対象の請求書を開く
2. 「発行する（適格請求書）」→ 確認ダイアログで「発行する」
   - 発行すると請求書番号（INV-YYYY-NNN）が採番され、内容は変更不可になります

### PDF ダウンロード

- **管理者向け（自分用）**: 詳細画面の「PDF をダウンロード」ボタン
- **クライアント向け URL**: 「クライアント向けリンクを生成」→ 表示された URL をコピーしてメール等で共有（7日間有効、ログイン不要）

### 入金記録

1. 発行済み請求書の詳細画面を開く
2. 入金フォームに金額・入金日・方法を入力
3. 「記録する」→ 確認ダイアログで「記録する」

## 5. セキュリティ設定

### .htaccess による保護（Apache）

`public_html/.htaccess`（既存の設定がある場合はマージ）:

```apache
# install.php へのアクセスを拒否
<Files "install.php">
    Require all denied
</Files>

# var/ ディレクトリを公開しない
<DirectoryMatch "var">
    Require all denied
</DirectoryMatch>
```

### JWT シークレット（必須）

`NENE2_LOCAL_JWT_SECRET` はログイントークン（オペレーター・サービス連携）の署名鍵です。
**本番（`APP_ENV=production`）では未設定だとアプリが起動を拒否します**（既知の既定値へ
フォールバックしません）。Web インストーラーは自動生成しますが、手動構築（Docker 等）の
場合は必ず強力な値を設定してください:

```bash
php -r "echo bin2hex(random_bytes(32));"
```

### リバースプロキシ配下の実クライアント IP

ログインのブルートフォース対策は**送信元 IP 単位**で機能します（アカウントロックアウト
DoS を避ける設計）。Nginx / ロードバランサーの背後では `REMOTE_ADDR` がプロキシの IP に
なり、全クライアントが 1 つの枠を共有してしまいます。プロキシ側で実クライアント IP を
アプリへ正しく渡す設定（信頼できるプロキシからの `X-Forwarded-For` 解決）を行ってください。

### メール送信（SMTP）の TLS

`MAIL_ENCRYPTION` が空の場合、SMTP は平文・無認証になります（ローカルの Mailpit 用）。
**本番では `tls` または `ssl` を設定**し、認証情報を必ず指定してください。

### 定期的なバックアップ

- `.env` ファイル（DB 接続情報・JWT シークレット）
- MySQL データベース全体

## 6. アップデート

1. 新しい ZIP を展開
2. `vendor/`, `src/`, `public_html/`（admin/ 以外）を上書きアップロード
3. `public_html/install.php` は**アップロードしない**（`.installed` が存在する場合は拒否されますが念のため）
4. `composer migrations:migrate` が実行できる環境の場合はマイグレーションを適用

> **Tier A の場合**: 新バージョンのマイグレーション SQL が提供される場合は phpMyAdmin 等で手動実行してください。

## 7. トラブルシューティング

### 503 エラー / 白い画面

- `APP_DEBUG=true` に変更してエラーを確認（確認後は必ず `false` に戻す）
- PHP エラーログを確認

### DB 接続エラー

- `.env` の `DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASSWORD` を確認
- ホスティング会社の MySQL ホスト名を確認（`localhost` でない場合あり）

### ログインできない

- メールアドレス・パスワードを確認
- `NENE2_LOCAL_JWT_SECRET` が `.env` に設定されているか確認
