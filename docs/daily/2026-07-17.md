## #38 — path テナンシーの cookie を slug スコープ Path で発行（PR #694・merged）

- **W2b 昇格ゲートの唯一の前提**。根本原因〔実測〕: `DemoSessionSeater` は初回 cookie を `/{base}/{slug}` にスコープするが、`RefreshHandler` の rotation が `BasePath::fromRequest`（OrgResolver が slug strip 済み＝install base のみ）で再発行 → `/{base}/auth`（slug なし）に着地 → 次の `/{slug}/auth/refresh` で rotated-away cookie を送り reuse 検知で family-burn。
- 修正: `BasePath` に `appBaseFromRequest()`＋app_base 属性、OrgResolver が strip 前に app base（install base + slug）を載せ、Login/Refresh/Logout が `appBaseFromRequest()` で cookie base を組む。非 path はフォールバックでバイト等価・`DemoSessionSeater` 無変更。`composer check` 全体緑（1007 tests）〔実測〕。統合リナ APPROVE で self-merge。
- スコープ: backend のみ。frontend の per-mode gate ON＋デプロイは施主 GO（施主キュー掲載済み〔伝聞〕）。
