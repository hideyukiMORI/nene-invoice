## Notable / 学び

- **前提の裏取りが効いた**: 引継ぎの「path テナンシー稼働」を鵜呑みにせず実コード（Router の anchored regex 照合・slug-strip 不在）を確認したことで、Phase 2 の真のスコープ（slug-strip ルーティング）を正しく捉えられた。
- **並行セッション衝突**: invoice 作業ディレクトリを本セッションと Opus リナ（S2）が共有。(1) S2 の path-repo commit が一時ローカル main に乗り、そこから切った docs ブランチの CI が `../NENE2 not found` で fail → origin/main へ rebase して是正。(2) `gh pr merge --delete-branch` が共有ツリーを S2 ブランチへ自動切替。→ 教訓: **origin/main 基点で切る**・commit 作業は**リンク worktree で隔離**（本 handover/日報もそれで作成）・`--delete-branch` は使わず手動掃除。memory `parallel-agent-collision-risk` に反映。
