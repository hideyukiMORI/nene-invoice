## NeNe Invoice v1.0.0

自己ホスト型の **見積・請求・入金管理** OSS（MIT）の最初の公開リリースです。
日本の SMB 向けに **適格請求書（インボイス）** に対応し、月額 SaaS なしで、
共有ホスティング（Tier A）または Docker/VPS（Tier B）に設置して使えます。

### できること

- **見積 → 請求 → 入金** の一連の管理と、**適格請求書対応 PDF** の発行（登録番号・税率区分・端数処理）
- **マルチテナント**・ロール別認可・httpOnly cookie によるサイレント再認証
- **定期請求**の自動下書き生成、**銀行入金の自動消込**（CSV 取込 → 名義照合 → 確認起票）
- **カード決済**（PAY.JP・hosted / SAQ-A）、**NeNe Clear 連携**（入金消込・督促の downstream）
- 管理 UI（React・日本語/English）、CSV 入出力、監査ログ、ダッシュボード
- **どこにでも設置**（ルート / サブディレクトリ / サブドメイン・再ビルド不要）、自己完結インストーラ

### はじめ方

```bash
git clone https://github.com/hideyukiMORI/nene-invoice.git
cd nene-invoice
docker compose up -d --build
# → http://localhost:8510（ログイン: admin@example.com / password123）
```

Tier A 共有ホスティング向けのリリース ZIP とインストーラも同梱しています。詳細は
[`README.md`](../../README.md) / [`docs/`](../) を参照してください。

### 信頼の柱

- 自己ホスト設置物に **テレメトリを一切載せません**（何も外部送信しません）。
- 会計・税務コンプライアンスを拘束ルールとして扱います（適格請求書・端数処理・採番・保存）。

## English summary

**NeNe Invoice v1.0.0** — the first public release of a self-hosted **quote,
invoice, and payment** OSS (MIT) for Japanese SMBs, with **qualified-invoice
(適格請求書)** PDFs, on shared hosting (Tier A) or Docker/VPS (Tier B), with no
monthly SaaS fee.

Highlights: quote-to-cash with compliant PDFs, multi-tenancy, silent
re-authentication, recurring billing, bank-transfer auto-reconciliation, hosted
card payments (PAY.JP), a NeNe Clear downstream link, a React admin UI (ja/en),
and install-anywhere deployment. Self-hosted installs carry **zero telemetry**.

Quick start: `docker compose up -d --build` → `http://localhost:8510`.
Full change list: [`CHANGELOG.md`](../../CHANGELOG.md).
