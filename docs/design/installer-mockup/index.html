セットアップウィザード — デザインハンドオフ資料 v1

これは何か

自己ホスト型の見積・請求 OSS「NeNe Invoice」を、共有ホスティング（ヘテムル / さくら等）にブラウザだけでインストールするウィザード（install.php）の全画面です。対象ユーザーは shell も CLI も使えない・「サブドメインって何?」レベルの中小企業の担当者。専門知識ゼロで完了できることが最重要。

この資料の HTML/CSS は中立なたたき台です。色・余白・タイポ・ロゴ・イラストは確定値ではありません。自由に再デザインしてください。確定しているのは「画面の順序・各画面の項目・コピー文言・出る/出ないの条件」です。

画面フロー

要件チェック → 1. データベース → 2. 管理者設定 → 3. 完了 → （ログイン画面へ）

ステッパーは「1 データベース / 2 管理者設定 / 3 完了」の 3 ステップ表記。要件チェックはステップ 1 の手前で毎回走る前提条件で、失敗時のみ専用表示になりブロックする（ステッパー上は独立カウントしない）。

収録画面・状態

  • 01 — 要件チェック
    PHP バージョン・拡張・書き込み権限・vendor の確認。合格（通過）/ 不合格（ブロック）の両状態。
  • 02 — ステップ1: データベース接続
    DB ホスト/ポート/名前/ユーザー/パスワード入力。空 / 入力済 / 接続エラー / 適用中（ローディング）。
  • 03 — ステップ2: 管理者設定
    会社名・管理者メール・パスワード。バリデーションエラー各種。
  • 04 — ステップ3: 完了
    成功画面。install.php 削除のセキュリティ警告・ログインへの導線。
  • 90 — 状態・エラー集
    インストール済みブロック(403)・DB エラー・各バリデーション・要件不合格を一覧。
  • 99 — コピーデック
    全文言（ラベル/プレースホルダ/補助/ツールチップ/エラー/ボタン）を一覧。翻訳・推敲の元データ。

デザインで特に考えてほしいこと

  • 非エンジニアでも迷わない導線・言葉づかい（専門用語にはツールチップ/補助文を必ず添える）。
  • DB 接続情報の在りかをどう案内するか（共有ホスティングのコントロールパネルから取得する旨）。ホストは localhost とは限らない点が最大の詰まりどころ。
  • ローディング状態（「接続テスト & スキーマ適用」は数秒かかりうる）。二重送信防止。
  • セキュリティ警告（完了後に install.php を削除）を見落とさせない設計。
  • エラーからの復帰（入力値は保持される前提）。
  • レスポンシブ（スマホのコントロールパネルからコピペする人も多い）。
  • ブランド/ロゴ・トーン（信頼感・適格請求書＝堅めの業務ソフト）。
