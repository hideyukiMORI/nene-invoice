要件チェックに失敗しました — 以下を解消してから再読み込みしてください。
  • ✓PHP 8.4 以上（現在: 8.4.22）
  • ✓PHP 拡張モジュール: pdo / pdo_mysql / mbstring / openssl / json
  • ✕var/ ディレクトリへの書き込み権限がありません
    解決: var/ のパーミッションを書き込み可（755/775）にしてください。
  • ✕ルートディレクトリへの書き込み権限がありません（.env ファイルを作成できません）
    解決: 展開先フォルダを書き込み可にしてください。インストール完了後は戻して構いません。
  • ✓vendor/ ディレクトリ（依存一式）が存在
実際のメッセージ（実装準拠）: 「PHP 8.4 以上が必要です（現在: …）」「PHP 拡張モジュール {名前} が見つかりません」「var/ ディレクトリへの書き込み権限がありません」「ルートディレクトリへの書き込み権限がありません（.env ファイルを作成できません）」「vendor/ ディレクトリが見つかりません。ZIP ファイルが完全に展開されているか確認してください」。
各項目に「どう直すか」の補助文／FAQ リンクを添えたい（対象ユーザーは権限変更の操作に不慣れ）。
合格時の案: このチェックリストを緑のチェックで一瞬見せてからステップ1へ、または無音でステップ1表示。前者は安心感、後者は最短。デザインで選択。
