# Local MCP Server

A **development-only** MCP (Model Context Protocol) server for NeNe Invoice, so
local agents can inspect the API through its documented boundaries. It consumes
NENE2's `Nene2\Mcp\LocalMcpServer` (ADR 0021).

> **Not a production backdoor.** It is never shipped in a release artifact, never
> auto-started, and binds no network port (it speaks JSON-RPC over stdio). It only
> reaches the **local** API base URL. See [ADR 0021](../adr/0021-local-mcp-server-consumer.md).

**Framework baseline:** NENE2 `docs/integrations/local-mcp-server.md` — this doc
records only the Invoice-specific wiring (port 8510, multi-tenant auth).

---

## Run it

Host-run (app on `:8510`):

```bash
php tools/local-mcp-server.php
```

Inside Docker Compose (target the `app` service):

```bash
docker compose run --rm -e NENE2_LOCAL_API_BASE_URL=http://app app php tools/local-mcp-server.php
```

It speaks newline-delimited JSON-RPC 2.0 on stdio: `initialize`, `tools/list`,
`tools/call`. STDOUT carries the protocol; STDERR carries the startup line and the
audit trail for state-changing calls (there are none in the read-only default).

Tools are the committed catalog `docs/mcp/tools.json` (generated by
`tools/generate-mcp-tools.php`, validated by `composer mcp`).

## Authentication (ADR 0021, option C — fail-closed)

Every tool except `getHealth` calls `/admin/*`, which passes through
`OrgGuardMiddleware` + capability checks, so it needs a bearer token. The server
obtains one in this order:

1. **`NENE2_LOCAL_MCP_BEARER`** — a pre-issued token, used verbatim. This is the
   default path; it derives no privilege from a secret. Issue one through the
   normal login flow or `php tools/issue-service-token.php` and pass it in.
2. **Dev-secret mint** — used **only** when `NENE2_ALLOW_DEV_SECRET=1` and
   `NENE2_LOCAL_JWT_SECRET` is set. The server mints a `{sub, role, org}` token
   through the app's own issuer, so it is signed with the exact secret the running
   API verifies with. Configure with `NENE2_LOCAL_MCP_ORG` (default `1`),
   `NENE2_LOCAL_MCP_SUB` (default `0`), `NENE2_LOCAL_MCP_ROLE` (default `admin`).
3. **Neither** — no token. Only `getHealth` works; every `/admin/*` tool returns
   401. This is the fail-closed default.

`role=admin` is the default because it is the **least-privilege non-superadmin
role that satisfies the whole read-only tool set**: `getCompanySettings` requires
`manage_company_settings`, which `member`/`viewer` lack. `admin` also has
`view_billing` for clients/quotes/invoices/payments.

## Exposure (read-only by default)

By default only the 10 `safety: read` tools are exposed. Set
`NENE2_LOCAL_MCP_INCLUDE_ADMIN=1` to also expose the 5 `admin` tools
(`listOrganizations`, `getOrganizationById`, `listUsers`, `getUserById`,
`listAuditLogs`). This widens **exposure only** — authorization stays at the API:
a token whose role does not satisfy an endpoint is still rejected there (e.g.
`listOrganizations` needs `superadmin`).

## The superadmin escape hatch (dev-only, triple opt-in)

Minting a `superadmin` token from the local secret is deliberately hard to reach.
It requires **all three** of:

```
NENE2_ALLOW_DEV_SECRET=1
NENE2_LOCAL_MCP_INCLUDE_ADMIN=1
NENE2_LOCAL_MCP_ROLE=superadmin
```

Requesting `role=superadmin` without `INCLUDE_ADMIN=1` is **refused at startup**
(the server exits non-zero with a clear message) rather than silently downgraded.
Do not relax the default. This exists only for local cross-tenant inspection.

## Smoke check

`tools/mcp-smoke.sh` pipes `initialize` + `tools/list` (+ an optional
`tools/call`) at the server. `initialize`/`tools/list` need no running API;
`tools/call` needs the app up (and MySQL + migrations for anything reading the DB).

```bash
docker compose up -d app
bash tools/mcp-smoke.sh                       # initialize + tools/list
bash tools/mcp-smoke.sh getHealth '{}'        # a read tool call
bash tools/mcp-smoke.sh listInvoices '{"limit":5}'
```
