# カード決済 会計・税務レビュー依頼書（税理士確認用・ドラフト）

> **位置づけ.** 本書は Issue #430 の成果物で、PAY.JP カード決済（受領側）の
> 会計・税務上の取り扱いを**税理士に確認していただくための依頼書**です。
> ADR 0012/0013 の方針に基づき、税理士サインオフは**リリースゲート**（実装は
> 先行済み、サインオフ前にリリースしない）。正本は
> [`../explanation/accounting-compliance.md`](../explanation/accounting-compliance.md)。
>
> 表中の金額・勘定科目は**たたき台**です。正しさのご判断・修正指示をお願いします。

## 0. 前提（現状の実装）

- 金額はすべて**整数（最小通貨単位＝円）**で保持（float 禁止・ADR 0004）。
- 消費税は**税率ごとに文書単位で1回**四捨五入（ADR 0004）。税率は 10% / 8% のみ。
- 請求書発行時に売上計上、入金は `payment` レコードで記録（複数入金で
  `partially_paid` → `paid` に遷移）。
- カード決済成功時、現状は **`payment` を総額（gross）で1本**起票している。
  手数料・差引入金・返金・チャージバックは**未モデル化**（本レビューの対象）。
- 決済代行は PAY.JP。決済手数料は概ね **2.59%**（プランによる）、売上振込手数料
  別途（〜¥250）。

## 1. 資金フロー（例：請求 ¥11,000 / 税10% / 手数料 2.59%）

```
請求総額            11,000   （税抜 10,000 + 仮受消費税 1,000）
  └ カード決済（PAY.JP）
       決済手数料   −285     （11,000 × 2.59% ≒ 285）
       差引          10,715
  └ 売上入金（振込）
       振込手数料   −250     （該当する場合）
       着金          10,465
```

- **論点**: 決済日と着金日にズレがある（決済確定＝webhook 受領日 vs 実際の入金日）。
  どの時点で何を計上するか（発生主義／実現主義の適用）をご確認ください。

## 3. 税理士への質問リスト

- **Q1 計上タイミング**: 売上は請求書発行日基準で計上し、入金は別途 `payment`
  で記録する現行設計で会計・税務上問題ないか。
- **Q2 決済手数料の消費税区分**: PAY.JP の決済手数料は**課税仕入**として処理して
  よいか（非課税・対象外の可能性は）。仕入税額控除の扱いは。
- **Q3 差引入金（net）の表現**: 入金は手数料控除後で着金するが、売掛金は**総額で
  消し込み**、手数料を別科目（支払手数料）で計上する #2 の方式で適切か。
  それとも純額（net）で記録すべきか。
- **Q4 返金時の適格返還請求書**: カード返金時に**適格返還請求書**の交付義務が
  生じるか。1万円未満の返還における交付義務免除の特例の適用範囲。
- **Q5 チャージバック**: 売上の取消として扱うか、貸倒れ（貸倒損失／貸倒引当）と
  して扱うか。消費税の取り扱い（売上対価の返還等 vs 貸倒れに係る税額控除）。
- **Q6 端数**: 手数料計算で生じる円未満端数の処理方法（切捨て等）。税率別 1 回
  丸め（ADR 0004）と干渉しないか。
- **Q7 保存要件**: カード決済に関する記録（charge id 等）の帳簿・証憑保存上の
  要件（電子帳簿保存法を含む）で、追加で保持すべき項目はあるか。

## 4. 適格請求書（インボイス）影響メモ

- カード決済でも、**交付する適格請求書の記載事項・税額計算は不変**である前提
  （発行体名・住所・`T`+13 桁登録番号・日付・税率別税抜額・税率別消費税額・合計・
  宛名）。これで問題ないか。
- 返金時の **適格返還請求書**の要否・記載事項（Q4 と連動）。
- 決済手数料は**当社の費用**であり、買い手に交付する請求書の税額計算には影響
  しない、という理解で正しいか。

## 5. データモデルへの含意（実装側メモ）

税理士回答により `payment` のデータモデルに手戻りが生じうる：

- 現状: `payment` = 総額1本（`amount_cents` / `external_reference` = charge id /
  `method='card'`）。手数料・返金・チャージバックのフィールドは未保持。
- 想定される変更例: 手数料額の保持、返金（マイナス `payment` or 別エンティティ）、
  チャージバック状態。**Q2/Q3/Q5 の回答後に確定**する。
